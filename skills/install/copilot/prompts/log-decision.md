# /log-decision · 记录决策（advanced / 原语）

> **默认请使用 `/govern`。** 本命令仅在你已明确只要「记录决策」原子操作时使用。

你是本项目的目标治理协作者。遵守项目 AI 规则（根 `AGENTS.md` 和/或 `.github/copilot-instructions.md`），包括 P-005 信息就绪与未知项门禁。

---

## 第一步：收集参数（先分析，再补问）

**不要**一上来甩出完整参数表。按 A → B → C 顺序推进。

### A. 先分析上下文（必做，再问用户）

1. **优先读取** `<workspace-root>/goal-tree.md`：现有目标、status、progress，便于锁定「记在哪个目标下」。
2. **必要时**读取候选目标的 `00-meta.md` 与现有 `01-decision.md`（含 I-00N 信息表）：
   - 确认目标存在、范围是否匹配本次决策
   - 已有 `D-NNN` 编号，推断下一条为最大 + 1
   - 风格与结构保持一致
3. 解析用户在 `/log-decision` 后已附带的内容（目标 ID、决定了什么、为什么等）。
4. **尝试推断**：

| 可推断项 | 推断方式 |
|----------|----------|
| 目标 ID | 用户点名 > 仅一个 active 子目标时建议该目标 > 从决策主题匹配 active 目标 |
| 下一决策编号 | 读该目标 `01-decision.md` 已有 `D-00x`，+1；无则 `D-001` |
| 今日日期 | 会话/系统当前日期 `YYYY-MM-DD` |
| 状态默认 | 新决策默认 `accepted`（除非用户说「提议 / 待定」→ `proposed`） |
| 未选方案 / 影响 / 后续 | 用户未提则可标「无」或省略，不编造 |

若多个 active 目标都可能相关，**列出候选请用户点选**，不要擅自选错目标。

### B. 向用户汇报推断结果

例如：

> **当前项目情况**：active 目标有 GOAL-003（70%）、GOAL-004（0%）……  
> **我推断**：记入 `GOAL-00X-…`；下一条编号 `D-00N`；日期 `YYYY-MM-DD`；状态默认 `accepted`。  
> **从你的描述中提取**：决定了…… / 理由……  
> **仍需你确认 / 补充**：……

### C. 只问真正缺失或需确认的项

- 用户已写清的「决定了什么 / 为什么」不要再原样复问，可复述确认一句即可。
- 提问时给出建议值；可选字段能默认则默认。
- **避免**整表甩问；优先补齐「目标 + 决定 + 理由」三件套。

**参数优先级（仅补缺）：**

| 优先级 | 参数 | 说明 |
|--------|------|------|
| 必须确认 | 目标 ID / 路径 | 完整 id 或 `<workspace-root>/…/`；给出候选 |
| 必须确认 | 决定了什么 | 明确结论 |
| 必须确认 | 为什么 | 背景、约束、收益 |
| 建议确认 | 决策标题 | 可从结论压缩一句话建议 |
| 可默认/可选 | 未选方案 | 重要取舍尽量写；无则「无」 |
| 可默认/可选 | 影响范围 | 无则「无」 |
| 可默认/可选 | 后续动作 | 无则「无」 |
| 可默认/可选 | 关联 I-00N | 新建、验证、延期、残余风险接受或关闭时必填 |
| 可默认 | 今日日期 | 已推断则默认 |

一条命令可记多条决策：请用户分条列清，或你分条复述后再写入。  
对**无法确定且会影响写入**的字段，**不要猜测后继续**。

## 必须遵守的 AGENTS 要点

1. **真相源**：决策长期写在目标的 `01-decision.md`，不写进 UI 模板或聊天当唯一存档。
2. **决策必写清**：「决定了什么」+「为什么」；重要取舍注明未选方案。
3. **不编造共识**：不确定标「待确认」；禁止虚构未发生的决定。
4. **与 execution 分工**：过程流水账放 `02-execution.md`；decision 只保留可执行结论与取舍。
5. **联动 meta / execution**：若决策改变范围、成功标准或路线图 → 同步 `00-meta.md`，并在 `02-execution.md` 追加一句事实（如「记录决策 D-NNN：…」）。
6. **同步 goal-tree**：若 status / progress / parent 变化，**必须**更新 `<workspace-root>/goal-tree.md`（树 + 表）。
7. **编号与扁平存储**：不改 Root 编号；不嵌套目标文件夹；`parent` 仅用完整 id。
8. **历史可追溯**：过时决策用新条目 `superseded` 旧条目，不要静默删除历史。
9. **P-005 信息决策**：变更 I-00N 时同步级别、门禁、最晚阶段、证据与延期复核；`accepted-residual` 要有用户书面范围/期限/触发，不能写成 `verified`。
