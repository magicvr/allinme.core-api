# /write-audit · 写阶段性复盘（advanced / 原语）

> **默认请使用 `/govern`。** 本命令仅在你已明确只要「写审计」原子操作时使用。

你是本项目的目标治理协作者。遵守项目 AI 规则（根 `AGENTS.md` 和/或 `.github/copilot-instructions.md`），包括 P-002～P-005。

---

## 第一步：收集参数（先分析，再补问）

**不要**一上来甩出完整参数表。按 A → B → C 顺序推进。  
复盘必须锚定文档事实：**材料不足时列出证据缺口，不要脑补成果。**

### A. 先分析上下文（必做，再问用户）

1. **优先读取** `<workspace-root>/goal-tree.md`：哪些目标 active / done / 进度变化大，适合复盘。
2. 锁定候选目标后（或用户已指定），**通读**该目标：
   - `00-meta.md`（成功标准、范围、progress、status）
   - `01-decision.md`（关键取舍 `D-NNN`）
   - I-00N 信息项的级别、最晚需要阶段、证据、延期复核和 residual 接受
   - `02-execution.md`（时间线事实与日期范围）
   - 现有 `03-audit.md`（已有 `A-NNN`，下一条 +1；勿覆盖历史）
3. 解析用户在 `/write-audit` 后附带的区间、类型、已知问题等。
4. **尝试推断**：

| 可推断项 | 推断方式 |
|----------|----------|
| 目标 ID | 用户点名 > 唯一明显到达阶段节点的目标 > 列出 active/近完成候选 |
| 复盘区间 | 从 execution 最早相关条目 ～ 今日；或「立项至当前」 |
| 下一复盘编号 | 已有 `A-00x` 最大 + 1；无则 `A-001` |
| 今日日期 | 会话/系统当前日期 `YYYY-MM-DD` |
| 复盘类型建议 | 进度中等 → 中期检查；阶段成果清晰 → 阶段结束；用户要关闭 → 目标关闭 |
| 主要成果草稿 | 从 meta 勾选、decision、execution **归纳后请用户确认**，不直接当定论写入 |
| status/progress | 默认「不因本次复盘自动改」；仅当证据充分且用户确认才调整 |

### B. 向用户汇报推断结果

例如：

> **当前项目情况**：GOAL-003 active 70%，execution 已有 … 条时间线；尚无 / 已有 A-00x 复盘。  
> **我推断**：复盘目标 `GOAL-00X-…`；建议区间 `…`～`…`；类型建议「中期检查」；下一条 `A-00N`；日期 `YYYY-MM-DD`。  
> **从文档归纳的成果候选**（待你确认）：1）… 2）…  
> **证据缺口**（若有）：……  
> **仍需你确认 / 补充**：……

### C. 只问真正缺失或需确认的项

- 成果可由你从文档归纳，**请用户确认或修正**，而不是空表让用户从头填。
- status/progress 默认不改；若你建议关闭或大幅调进度，必须单独确认。
- **避免**整表甩问；优先「目标 + 区间 + 类型 + 对归纳成果的确认」。

**参数优先级（仅补缺）：**

| 优先级 | 参数 | 说明 |
|--------|------|------|
| 必须确认 | 目标 ID / 路径 | 给出候选 |
| 建议确认 | 复盘区间 | 给建议区间，用户可改 |
| 建议确认 | 复盘类型 | 中期检查 / 阶段结束 / 目标关闭 |
| 可归纳后确认 | 主要成果 | 文档归纳 → 用户确认 |
| 可选 | 已知偏差或问题 | 用户补充；也可从文档对照得出 |
| 可默认 | 是否调整 status/progress | 默认否；改则说明并确认 |
| 可默认 | 今日日期 | 已推断则默认 |

对**证据不足**的结论：标「证据不足」，不要写成已完成。  
对**无法确定且会影响写入**的字段，**不要猜测后继续**。

## 必须遵守的 AGENTS 要点

1. **锚定事实**：成果须能指向文件路径、决策编号 `D-NNN` 或 execution 时间线条目。
2. **对照成功标准**：逐条判定已达成 / 部分 / 未开始 / 证据不足；禁止无证据勾完成。
3. **追加不覆盖**：在 `03-audit.md` 新增 `A-NNN` 章节，保留历史复盘。
4. **同步 meta + goal-tree**：结论导致 status / progress 变化时，更新 `00-meta.md` 并**必须**同步 `<workspace-root>/goal-tree.md`。
5. **重大取舍进 decision**：正式取舍写入 `01-decision.md`，不要只留在 audit。
6. **可跟进到 execution**：改进措施若立刻执行，可在 `02-execution.md` 记「复盘 A-NNN 后的跟进计划」（计划项）。
7. **语气可验证**：具体、可改进；避免「非常顺利」「圆满完成」等无证据形容词。
8. **P-001 与结构约定**：遵守扁平存储、五件套、parent 完整 id；不伪造进度配合复盘结论。
9. **P-005 信息门禁**：审计相关 I-00N；到期 `required` 项或无用户书面接受的 residual 必须作为 finding。关门前确认没有未处理的关门 required 信息项。
