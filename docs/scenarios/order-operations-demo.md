# 订单运营完整演示

## 1. 目标

本场景把 Schema-UI 的看板、搜索表格、联动表单、行级 Action 和上传能力映射到同一套可持久化订单数据。页面配置来自后端，业务请求经过真实认证、授权、校验和 SQLite 事务；重启保留操作结果，reset/seed 可恢复基线。

本文件描述业务验收，不复制协议仓中的页面 YAML。页面结构与执行语义以 [`schema-ui-docs` 场景](../../../schema-ui-docs/docs/05-scenarios/README.md) 为准。

## 2. 参与者

| 角色 | 场景职责 |
|---|---|
| viewer | 查看经营看板、订单和已绑定附件 |
| operator | 创建/编辑订单、推进履约、取消订单、上传附件、发起退款 |
| approver | 查看退款队列并审批或拒绝他人申请 |
| admin | 拥有全部首批业务能力；账号管理属于后续目标 |

所有参与者先通过本地账号登录获取 JWT。切换角色必须重新登录对应账号，不接受客户端声明角色。

阶段四的退款队列仅供 `approver`/`admin` 审批使用。`operator` 发起退款后只通过订单 `paymentStatus`、`availableRefundAmount` 和 `canRequestRefund` 观察聚合结果，不提供逐笔退款历史；若后续产品要求申请人逐笔跟踪，必须先扩展目标 API 和场景，再实现页面，阶段六不得临时放宽审批队列角色。

## 3. 主流程

本节使用的 page ID 均为 provisional 叙述标识，最终值随阶段六页面文件和目标 API 一起冻结。

1. operator 登录并打开 `order-editor` 页面，从后端取得已校验的 Schema-UI JSON。
2. operator 填写客户和订单项；字段 Reaction 控制相关输入，但服务端仍执行完整校验。
3. operator 上传 PDF/图片附件，取得临时附件 ID，再使用幂等键创建订单并绑定附件。
4. operator 在 `order-list` 页面按客户或状态搜索；表格从服务端分页，并显示当前行可执行 Action。
5. operator 依次确认、开始履约、发货和完成订单；每次 Action 提交当前版本，成功后刷新列表。
6. 对 seed 中的已支付订单，operator 发起部分退款；重复提交同一幂等键只产生一条退款。
7. approver 登录 `refund-queue`，审批该退款；服务端禁止申请人审批自己的申请。
8. viewer 打开 `order-dashboard`，看板从更新后的订单和退款数据计算指标与趋势。
9. viewer 从订单详情下载已绑定附件；无订单查看权限、未认证或访问未绑定附件时被拒绝。

## 4. 关键失败路径

| 条件 | 预期结果 |
|---|---|
| 错误密码、禁用账号 | 返回统一认证失败语义，不泄露账号存在性 |
| viewer 调用写 Action | 返回权限不足语义，数据库不变化 |
| 旧 `version` 推进状态 | 返回版本冲突语义，客户端刷新资源后重试 |
| 非法状态转换 | 返回业务状态冲突语义，不能依赖隐藏按钮防止调用 |
| 同一幂等键配不同请求体 | 返回幂等冲突语义 |
| 退款超过可退金额 | 返回可定位到金额字段的校验失败语义 |
| 申请人审批自己的退款 | 返回权限不足语义 |
| 超限、伪装类型或危险文件名 | 返回上传拒绝语义，不遗留最终文件 |
| 绑定他人的临时附件 | 返回权限不足语义 |
| JWT 过期、篡改或 session 已撤销 | 返回需要重新认证语义 |

每类失败最终必须返回稳定机器码；机器码名称以目标 API 在实现阶段的收敛结果为准，本场景不构成名称兼容承诺。

## 5. 数据与恢复

seed 覆盖四种角色、全部订单状态、已支付与部分退款订单、待审批退款及可下载附件。测试和演示不得依赖运行顺序；每个自动化套件使用独立临时数据库和文件目录。

reset 命令只在开发模式工作，执行时清理 SQLite 数据、附件目录和历史 session，再重新运行 migrations 与 seed。生产模式调用 reset 必须直接失败。

## 6. 验收证据

- API 集成测试覆盖主流程与全部失败路径；
- repository 测试验证事务、外键、乐观锁和幂等记录；
- 页面 YAML 通过固定 Schema-UI 版本的 L0-L4 校验；
- 前端加载后端页面 JSON 后完成搜索、提交、Action reload、上传和看板渲染；
- reset/seed 后重复演示得到相同初始业务状态。
