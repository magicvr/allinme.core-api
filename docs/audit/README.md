# 后端审计记录

审计采用 review、checklist、可选 plan 的闭环：发现问题后创建，修复时逐项勾选，全部验证并经用户明确确认后移入 [`archived/`](./archived/README.md)。

## 编号规则

- 文件名：`NNNN-YYYY-MM-DD-{review,checklist,plan}.md`。
- 创建前同时扫描活跃区和归档区，取最大编号 + 1；编号永不重置或复用。
- 问题使用全局递增 `V<n>`；首次审计从 `V1` 开始，后续同时扫描活跃和归档正文。
- 未证实的推测列为待确认，不分配缺陷编号。

## 归档规则

- checklist 全部为 `[x]`，并记录实际验证结果；
- 向用户汇报修复、未执行项和剩余风险；
- 只有用户明确确认后才移动整套文件并更新归档索引；
- 历史审计不是当前规范，当前规则以 [`../00-overview.md`](../00-overview.md) 和源码为准。
