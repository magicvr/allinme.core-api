# 阶段三：订单查询与履约 Checklist

配套计划：[阶段三订单查询与履约开发计划](./0003-2026-07-12-plan.md)。事实源按关注点拆分：领域规则见[领域模型](../05-domain-model.md)，阶段三详细 HTTP/CORS 契约见 plan §3-§4，endpoint 范围见[目标 HTTP API](../03-http-api-target.md)，可执行证据只记录在本 checklist。

统一测试规范：测试名、helper、executor、barrier、decoder 和连接中止机制均可替换；并发/连接测试设置不超过 10 秒的 deadline，并在 cleanup 释放 goroutine、channel、事务、DB、server 和临时目录；日志和失败输出不得包含 Authorization、token、密码、请求 body 或幂等 snapshot。证据缺失、只依赖人工观察、自动断言失败、超时、goroutine 泄漏、未关闭资源或临时目录残留，均直接阻断对应 gate。等价实现必须记录“原契约、替代证据、等价理由、reviewer、日期”。每项只勾选一个可观察结果，证据使用 `test`、`review`、`doc` 或 `smoke`，并记录实际命令、测试名或链接。
