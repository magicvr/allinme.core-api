# 阶段四：退款与经营看板 Checklist

配套计划：[阶段四退款与经营看板开发计划](./0004-2026-07-13-plan.md)。领域事实以[领域模型](../05-domain-model.md)为准，endpoint 范围以[目标 HTTP API](../03-http-api-target.md)为准，验证启用状态以[验证规则](../04-validation.md)为准。本 checklist 只记录实施门禁和实际 Evidence。

统一要求：每项完成后在下一行填写日期、类型、命令/测试、revision 或 smoke 结果。带 `-run` 的命令只用于开发加速，不能替代对应包级硬门禁。P0、M1/M2/M3 的功能正确性、并发、幂等、HTTP 契约和包级 test/race 是开发硬门禁；schema-only/旧 binary 恢复、query plan、完整 smoke、远端 CI 和归档记录是发布证据，可在对应功能切片稳定后集中完成，但发布前不得省略。并发测试必须设置 deadline 并释放 goroutine、事务、DB、server 和临时目录；日志和失败输出不得包含 Authorization、token、密码、完整请求 body 或幂等 snapshot。
